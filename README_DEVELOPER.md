# دليل تطوير الموقع الإلكتروني: المرجع الشامل MedRes

هذا الدليل مخصص للمطورين (أو نماذج الذكاء الاصطناعي) الذين يرغبون في فهم، تعديل، أو تطوير الموقع الإلكتروني "المرجع الشامل MedRes". الموقع هو دليل إرشادي طبي شامل، مصمم لطلاب الطب والباحثين، ويقدم معلومات تفاعلية حول البحث العلمي الطبي.

## 1. نظرة عامة على المشروع

الموقع مبني باستخدام أحدث تقنيات الواجهة الأمامية لضمان تجربة مستخدم سريعة، تفاعلية، ومتجاوبة. يدعم الموقع اللغة العربية (RTL) بشكل كامل، ويحتوي على وضعين للعرض (فاتح وداكن).

## 2. التقنيات المستخدمة

-   **React:** مكتبة JavaScript لبناء واجهات المستخدم.
-   **TypeScript:** مجموعة شاملة من JavaScript تضيف أنواعاً ثابتة لتحسين قابلية الصيانة وقابلية التوسع.
-   **Tailwind CSS:** إطار عمل CSS قائم على الأدوات المساعدة لتصميم سريع ومتجاوب.
-   **Vite:** أداة بناء سريعة توفر تجربة تطوير محسنة.
-   **pnpm:** مدير حزم سريع وفعال للمشروع.

## 3. هيكل المشروع

```
medical_site/
├── client/                 # مجلد تطبيق الواجهة الأمامية (React)
│   ├── public/             # الملفات الثابتة (الصور، القوالب، إلخ)
│   │   ├── templates/      # قوالب Word و Excel
│   │   └── ...
│   ├── src/                # الكود المصدري للتطبيق
│   │   ├── assets/         # الأصول الأخرى (مثل الأيقونات)
│   │   ├── components/     # المكونات القابلة لإعادة الاستخدام (مثل Navbar, Footer, AccordionSection)
│   │   ├── pages/          # صفحات الموقع الرئيسية (Home, AdditionalSections, MissingSections, AdvancedSections)
│   │   ├── App.tsx         # المكون الرئيسي للتطبيق
│   │   ├── index.css       # التنسيقات العالمية (Tailwind base, components, utilities)
│   │   └── main.tsx        # نقطة دخول التطبيق
│   ├── index.html          # ملف HTML الرئيسي
│   ├── package.json        # تعريف المشروع واعتماداته
│   ├── pnpm-lock.yaml      # ملف قفل الحزم لـ pnpm
│   ├── tsconfig.json       # إعدادات TypeScript
│   └── vite.config.ts      # إعدادات Vite
└── README_DEVELOPER.md     # هذا الملف
```

## 4. إعداد بيئة التطوير

لإعداد المشروع وتشغيله محلياً، اتبع الخطوات التالية:

1.  **استنساخ المستودع (Clone the repository):**
    ```bash
    git clone <repository_url>
    cd medical_site
    ```
    (ملاحظة: في بيئة الساندبوكس، الملفات موجودة بالفعل في `/home/ubuntu/medical_site`)

2.  **الانتقال إلى مجلد العميل (Navigate to client directory):**
    ```bash
    cd client
    ```

3.  **تثبيت الاعتمادات (Install dependencies):**
    تأكد من تثبيت `pnpm` على نظامك. إذا لم يكن مثبتاً، يمكنك تثبيته عبر `npm install -g pnpm`.
    ```bash
    pnpm install
    ```

## 5. تشغيل المشروع

بعد تثبيت الاعتمادات، يمكنك تشغيل خادم التطوير:

```bash
pnpm dev
```

سيتم تشغيل التطبيق على `http://localhost:3000` (أو منفذ آخر متاح). يمكنك الوصول إليه عبر المتصفح.

## 6. بناء المشروع للإنتاج (Building for Production)

لإنشاء نسخة جاهزة للإنتاج من التطبيق:

```bash
pnpm build
```

سيتم إنشاء ملفات الإنتاج في مجلد `dist/` داخل مجلد `client/`.

## 7. الميزات الرئيسية والمكونات الهامة

-   **`EvidencePyramid.tsx`:** مكون هرم الأدلة الطبية التفاعلي مع تلميحات (Tooltips) وروابط انتقال سلس.
-   **`SampleSizeCalculator.tsx`:** حاسبة حجم العينة المتقدمة مع أيقونات معلومات وتلميحات لكل معلمة.
-   **`ScrollReveal.tsx`:** مكون مخصص لتطبيق حركات الظهور (Fade-In-Up) عند التمرير، مع تحسينات لضمان توافقية RTL.
-   **`index.css`:** يحتوي على التنسيقات العالمية، بما في ذلك فئات Tailwind المخصصة للحركات وتأثيرات التحويم (مثل `category-card`, `btn-interactive`, `icon-interactive`).
-   **`AdditionalSections.tsx`، `MissingSections.tsx`، `AdvancedSections.tsx`:** تحتوي على معظم محتوى الموقع المنظم في أقسام قابلة للطي (AccordionSections).
-   **دعم RTL والوضع الليلي:** تم تصميم الموقع بالكامل لدعم اللغة العربية والتبديل بين الوضعين الفاتح والداكن بسلاسة.

## 8. ملاحظات هامة للمطورين

-   **تجنب الحركات الأفقية:** نظراً لدعم RTL، يفضل استخدام حركات رأسية (مثل `translateY`) للحفاظ على التخطيط ومنع مشاكل التمرير الأفقي.
-   **فئات Tailwind المخصصة:** استخدم الفئات المخصصة الموجودة في `index.css` للحفاظ على الاتساق البصري وتجنب تكرار الكود.
-   **فحص المتصفح:** استخدم أدوات المطور في المتصفح للتحقق من الأداء، وتوافقية RTL، وتأثيرات الوضع الليلي عند إجراء التعديلات.
