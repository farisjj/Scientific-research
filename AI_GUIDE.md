# دليل الذكاء الاصطناعي لمشروع دليل البحث العلمي الطبي (MedRes)

## نظرة عامة على المشروع
هذا المشروع عبارة عن موقع إلكتروني إرشادي تفاعلي باللغة العربية مخصص لطلاب الطب والباحثين في المجال الطبي. يوفر الموقع معلومات شاملة حول منهجيات البحث العلمي، أنواع الدراسات، نصائح عملية، قوالب جاهزة للتحميل، وأدوات مساعدة مثل حاسبة حجم العينة.

## التقنيات المستخدمة
المشروع مبني باستخدام أحدث تقنيات الواجهة الأمامية:
-   **React:** مكتبة JavaScript لبناء واجهات المستخدم.
-   **TypeScript:** مجموعة فرعية من JavaScript تضيف أنواعًا ثابتة.
-   **Vite:** أداة بناء سريعة لتطوير الويب.
-   **Tailwind CSS:** إطار عمل CSS قائم على الأدوات المساعدة لتصميم سريع الاستجابة.
-   **pnpm:** مدير حزم سريع وفعال.
-   **Framer Motion:** مكتبة لإضافة حركات وتأثيرات سلسة.

## إرشادات الإعداد والتشغيل للذكاء الاصطناعي

للقيام بأي تعديلات أو تشغيل للمشروع، اتبع الخطوات التالية:

1.  **المتطلبات الأساسية:**
    *   تأكد من تثبيت Node.js (الإصدار 18 أو أحدث).
    *   تأكد من تثبيت pnpm. إذا لم يكن مثبتًا، يمكنك تثبيته باستخدام `npm install -g pnpm`.

2.  **تثبيت التبعيات:**
    انتقل إلى مجلد المشروع الرئيسي (`medical_site/medical_site`) وقم بتثبيت جميع التبعيات:
    ```bash
    cd /home/ubuntu/medical_site/medical_site
    pnpm install
    ```

3.  **تشغيل خادم التطوير (Development Server):**
    لتشغيل الموقع في وضع التطوير مع خاصية التحديث السريع (Hot Reloading):
    ```bash
    cd /home/ubuntu/medical_site/medical_site
    pnpm dev
    ```
    سيتم تشغيل الخادم عادةً على `http://localhost:3000`.

4.  **بناء المشروع للإنتاج (Production Build):**
    لإنشاء نسخة محسّنة وجاهزة للإنتاج من الموقع:
    ```bash
    cd /home/ubuntu/medical_site/medical_site
    npx vite build
    ```
    سيتم وضع الملفات المبنية في مجلد `dist/public`.

5.  **خدمة الملفات المبنية (Serving Production Build):**
    لخدمة الملفات المبنية بشكل ثابت (على سبيل المثال، باستخدام خادم HTTP بسيط):
    ```bash
    cd /home/ubuntu/medical_site/medical_site
    python3 -m http.server 8080 --directory dist/public
    ```
    سيتم تشغيل الخادم على `http://localhost:8080`.

## المكونات والميزات الرئيسية

*   **`ScrollReveal`:** مكون React (موجود في `client/src/components/ScrollReveal.tsx`) يستخدم `IntersectionObserver` لإضافة حركات دخول سلسة (fade-in-up) للعناصر عند ظهورها في مجال الرؤية. يتم استخدامه في `Home.tsx` لتطبيق الحركات على الأقسام الرئيسية.
*   **`SearchBar`:** مكون البحث (موجود في `client/src/components/SearchBar.tsx`) يوفر وظيفة البحث داخل الموقع ويحتوي على قائمة بالكلمات المفتاحية والأقسام المستهدفة.
*   **`ResearchTemplates`:** مكون (موجود في `client/src/components/ResearchTemplates.tsx`) يعرض بطاقات لقوالب بحثية قابلة للتحميل (مثل Case Report و Data Extraction Sheet). تم تعديل هذا المكون لضمان تحميل الملفات بالامتدادات الصحيحة (`.docx`, `.xlsx`).
*   **`Home.tsx`:** صفحة الهبوط الرئيسية التي تجمع جميع الأقسام والمكونات الفرعية. هي النقطة المركزية لتطبيق حركات الدخول على مستوى القسم.

## ملاحظات هامة للذكاء الاصطناعي

*   **التعديلات على الـ `download` attribute:** عند التعامل مع روابط التحميل، تأكد دائمًا من أن الـ `download` attribute في وسم `<a>` يتضمن اسم الملف الكامل مع امتداده (مثال: `download="Case_Report_Template.docx"`) لضمان تحميل الملف بالصيغة الصحيحة.
*   **إضافة الحركات:** عند إضافة حركات جديدة، استخدم مكون `ScrollReveal` أو أنماط CSS مشابهة لضمان الاتساق والأداء. تجنب الحركات المبالغ فيها أو التي تؤثر على تجربة المستخدم.
*   **التجاوبية (Responsiveness):** الموقع مصمم ليكون متجاوبًا. عند إجراء أي تعديلات على التصميم، تأكد من اختبارها على أحجام شاشات مختلفة (خاصة الجوال) لتجنب مشاكل التجاوز الأفقي أو العناصر غير المرئية.
*   **محاذاة النصوص المختلطة:** الموقع يدعم اللغة العربية بشكل أساسي. عند دمج نصوص عربية وإنجليزية في نفس السطر أو الفقرة، تأكد من الحفاظ على المحاذاة الصحيحة من اليمين إلى اليسار (RTL) لضمان سهولة القراءة.
